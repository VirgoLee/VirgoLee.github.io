Hexo个人博客之转移（四）

到此为止，我们已经完成了差不多所有的步骤。

  • 1.搭建博客
  • 2.优化主题
  • 3.部署收录

新问题：

​ 现在博客只能在自己的电脑上更新，如果换电脑了就很麻烦。配置文件主题什么的都要重新弄。

所以网上找了找多台电脑同时操作的办法，我们可以利用Github的分支功能。将博客文件夹下所有文件全push到Github。这样换电脑后直接pull就可以了。

1.新建分支

在Github的lillusory.github.io（hexo仓库）上新建一个分支，例如Hexo，并切换到该分支，并在该仓库Settings->Branches->Default branch中将默认分支设为Hexo.Hexo分支是博客的开发环境，用来写博客，保存原始文件。master分支用于显示，保存生产的静态文件。

新建分支后将博客目录下的所有文件上传到该分支，注意由于一个git仓库中不能包含其他仓库，所以需要删除掉主题文件夹中的.git目录。如果按照前面的博文添加了背景，则需要删掉站点目录\source\lib\canvas-nestt文件夹中的.git目录。以后需要更新主题时，可以先克隆到本地在复制到相应目录，或者不删除.git目录，直接修改一些名字也行。

2.写博客

在本地对博客进行修改（添加新博文、修改样式等等）后，通过下面的流程进行管理。

  • 依次执行git add .、git commit -m "这里写备注"、git push origin 这里写分支名字指令将改动推送到GitHub（此时当前分支应为hexo）。
  • 然后才执行hexo g -d发布网站到master分支上。

3.博客转移

当重装电脑之后，或者想在其他电脑上修改博客，可以使用下列步骤：

  • 使用git clone git@github.com:lillusory/lillusory.github.io.git拷贝仓库（默认分支为hexo）；//修改成自己的
  • 在前面克隆下的项目中安装插件,执行命令npm install hexo、npm install、npm install hexo-deployer-git（记得，不需要hexo init这条指令）。
